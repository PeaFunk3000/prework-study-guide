HTML

  • The head element contains information about the webpage.
  • Meta and Title elements are examples that are nested inside the head element
  • The body element represents the visible content shown to the user.
  • Section elements are semantic elements that define a section in a document that contains thematic content. In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage
  • Not all elements are semantic. It is still common to see non-semantic elements, but best practice is to use semantic elements where possible.
  • The h1, h2, h3, h4, h5 and h6 elements represent the level of heading a given text block represents.
  • Elements that are nested inside other elements are called child elements. Elements that hold the children are parent elements.
  • The p element represents a paragraph or a block of text.
  • The ul, ol and li elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.
  • The img element contains imformation about images that are displayed on the webpage.
  • The img element can contain various attributes; the src attribute is the most important because it defines the location of the image.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element (anchor) creates links to webpages. The href attribute points to the URL for the link.

CSS Rule

CSS Box Model

    • A margin indicates how much space we want around the outside of an element.
    • A padding indicates how much space we want around the content inside an element.
    • Combining two selectors, eg. h1,h2, into a single CSS rule with a comma, allows the declarations to affect both selectors.
    • This helps us write Do Not Repeat Yourself (DRY) code.
    • In CSS we broadly have two types of boxes — block boxes and inline boxes.

Git

It's important to frequently sync the local version of your code with the remote version in GitHub.

  • It is a good habit to always pull the base branch into your feature branch before pushing up any code using git pull origin main

Git Flow

  • Add, commit, and push our work to current feature (branch)
  • Open pull request to merge changes with the main branch

Cheat Sheet

  • git status: checks what branch we are currently on
  • git pull: fetch and download content from a remote repository and immediately update the local repository to match that content
  • git log: shows all commits in the current branch's history
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stages all files in the working branch
  • git commit -m: commits changes made, -m flag to associate a message with the commit
  • git log: shows the commit history for the currently active branch

JavaScript

  • JavaScript is an object-oriented programming (OOP) language.
  • Conditional logic and arrays—can be applied to many other in-demand OOP languages.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Note that whereas the style.css file is linked within the head element, the script.js file is linked within the body element.
  • Arrays are useful tools for storing multiple items with different data types in a single variable, but not all arrays must have multiple data types.
  • Arrays are zero-indexed and sequential. This means that the first item in any array has an index of 0, not 1.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Functions are the building blocks of JavaScript. When we create a function, we are defining a block of code that performs a specific task and giving it a unique name.
  • Just defining a function does not mean it will execute in our code automatically. To execute a function, we must call it in our code.
